# CPU vs GPU

## 핵심 개념

- **CPU (Central Processing Unit)**: 범용 연산 처리 장치. 복잡한 연산을 **순차적**으로 빠르게 처리하는 데 최적화되어 있음
- **GPU (Graphics Processing Unit)**: 단순한 연산을 **병렬**로 대량 처리하는 데 최적화된 장치. 원래 그래픽 렌더링용이었지만 현재는 AI/딥러닝 연산에도 핵심적으로 사용됨

## 비교

| 항목        | CPU                    | GPU                         |
| ----------- | ---------------------- | --------------------------- |
| 코어 수     | 수 ~ 수십 개           | 수천 ~ 수만 개              |
| 처리 방식   | 순차 처리              | 병렬 처리                   |
| 적합한 작업 | 복잡한 로직, 분기 처리 | 단순 반복 연산 (행렬 곱 등) |
| 활용 예시   | OS, 게임 로직          | 딥러닝 학습, 렌더링         |

## AI분야에서 활용

딥러닝은 결국 행렬 곱의 반복인데, 이게 수천 개의 코어로 동시에 계산하면 CPU 대비 압도적으로 빠름

## 게임 개발에서의 역할 분담

게임 엔진은 CPU와 GPU가 각자 잘하는 걸 나눠서 처리함

담당처리 내용
**CPU** 게임 로직, AI 판단, 물리 연산, 입력 처리, 네트워크
**GPU** 메시 렌더링, 셰이더 연산, 조명 계산, 포스트 프로세싱

Draw Call: CPU가 GPU에게 "이거 그려라"고 명령을 보내는 것. Draw Call이 많아질수록 CPU→GPU 통신이 늘어나 병목 발생
Unity 기준: 게임 로직은 C# 스크립트(CPU), 렌더링은 GPU가 담당. Update()가 CPU에서 돌고, 화면에 뿌리는 건 GPU가 처리
Compute Shader: GPU를 렌더링 외 용도(파티클, 군중 시뮬레이션 등)로 쓸 때 활용 → GPU 병렬성을 게임 로직에도 끌어다 씀
