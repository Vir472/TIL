# 데이터 영역 (Data Segment)

## 핵심 개념

- 프로세스 메모리의 4가지 영역(Code, Data, Heap, Stack) 중 하나
- 전역변수(Global Variable)와 정적변수(Static Variable)가 저장되는 공간
- 프로그램이 시작될 때 할당되고, 종료될 때까지 메모리에 유지됨 (Heap/Stack처럼 동적으로 생기고 사라지지 않음)

## 더 세분화하면 — BSS와 Data로 나뉨

데이터 영역은 사실 내부적으로 두 부분으로 나뉨

```
┌─────────────────────┐
│   Data (초기화됨)     │  ← 초기값이 있는 전역/정적 변수
├─────────────────────┤
│   BSS (초기화 안 됨)  │  ← 초기값 없는 전역/정적 변수 (0으로 자동 초기화)
└─────────────────────┘
```

| 영역                          | 저장 내용                      | 예시                             |
| ----------------------------- | ------------------------------ | -------------------------------- |
| Data                          | 초기값이 명시된 전역/정적 변수 | `static int score = 100;`        |
| BSS (Block Started by Symbol) | 초기값 없는 전역/정적 변수     | `static int score;` (자동으로 0) |

- BSS 영역은 실행 파일에 실제 값을 저장할 필요가 없어서 (어차피 0이라) 실행 파일 크기를 줄여줌
- 프로그램 로드 시 OS가 BSS 영역을 0으로 채워줌

## 다른 메모리 영역과 비교

| 영역     | 저장 내용                | 생존 기간              | 관리 방식                |
| -------- | ------------------------ | ---------------------- | ------------------------ |
| Code     | 실행 코드                | 프로그램 시작~종료     | OS, 읽기 전용            |
| **Data** | **전역/정적 변수**       | **프로그램 시작~종료** | **자동 (프로그램 단위)** |
| Heap     | 동적 할당 객체           | 할당~해제(GC) 시점까지 | 수동 또는 GC             |
| Stack    | 지역변수, 함수 호출 정보 | 함수 호출~반환까지     | 자동 (함수 단위)         |

데이터 영역의 핵심 특징은 **생존 기간이 프로그램 전체**라는 것. Stack처럼 함수 끝나면 사라지지도 않고, Heap처럼 명시적으로 해제하지 않아도 됨 — 대신 프로그램이 끝날 때까지 계속 메모리를 차지함

## C# 관점에서

```csharp
public class GameManager
{
    static int totalScore = 0;        // Data 영역 (초기값 0이지만 명시했으므로 Data)
    static List<string> playerNames;  // 참조 자체는 Data 영역, 실제 List 객체는 Heap

    void Start()
    {
        int localScore = 10;          // 이건 Stack
    }
}
```

- `static` 키워드가 붙은 필드는 데이터 영역에 저장됨 (정확히는 .NET의 경우 "정적 필드 영역"이라는 별도 개념으로 CLR이 관리하지만, 개념적으로 C/C++의 데이터 영역과 같은 역할)
- 클래스가 아니라 **타입 자체**에 종속되므로, 인스턴스를 몇 개를 만들어도 static 변수는 단 하나만 존재

## 게임 개발 관점에서

- **싱글톤 패턴과 연결됨**: Unity에서 흔히 쓰는 `GameManager.Instance` 같은 싱글톤은 static 필드로 구현되는 경우가 많음 → 데이터 영역에 올라가서 게임 전체에서 하나의 인스턴스로 유지됨
- **주의할 점 — 메모리 누수 위험**: static 변수는 프로그램 종료까지 절대 해제되지 않음. 큰 데이터를 static으로 들고 있으면 게임 내내 메모리를 차지함 (특히 모바일 게임에서 치명적)
- **씬 전환 시 데이터 유지**: Unity에서 씬(Scene)이 바뀌면 일반 오브젝트는 사라지지만, static 변수는 데이터 영역에 그대로 남아있어서 씬 간 데이터 전달용으로 활용 가능 (`DontDestroyOnLoad`보다 가벼운 대안으로 쓰이기도 함)

```csharp
// 게임 상수는 이렇게 데이터 영역에 고정해두는 게 좋음
public static class CombatConstants
{
    public static readonly float CoinSuccessRate = 0.5f;
    public static readonly int MaxSpeed = 10;
}
```
