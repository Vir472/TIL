# WAV (Waveform Audio File Format)

## 핵심 개념

- **Waveform Audio File Format**의 약자. 확장자는 `.wav`
- Microsoft와 IBM이 1991년 만든 오디오 포맷
- 기본적으로 **비압축(Uncompressed)** — 오디오 샘플 데이터를 있는 그대로 저장
- 이미지의 BMP와 비슷한 위치 — 무손실이지만 파일이 큼

## 파일 구조 — RIFF 청크 기반

```
┌──────────────┐
│  RIFF 헤더   │  ← "RIFF" 식별자 + 파일 크기
├──────────────┤
│  fmt 청크    │  ← 샘플레이트, 비트 깊이, 채널 수 등 포맷 정보
├──────────────┤
│  data 청크   │  ← 실제 오디오 샘플 데이터
└──────────────┘
```

BMP처럼 구조가 단순해서 파서 구현이 쉬움

## 다른 오디오 포맷과 비교

| 항목        | WAV                      | MP3           | OGG           | FLAC          |
| ----------- | ------------------------ | ------------- | ------------- | ------------- |
| 압축        | 없음 (기본)              | 손실          | 손실          | 무손실        |
| 파일 크기   | 매우 큼                  | 작음          | 작음          | 중간          |
| 음질        | 완전 무손실              | 손실 있음     | 손실 있음     | 무손실        |
| 인코딩 속도 | 빠름                     | 느림          | 느림          | 중간          |
| 용도        | 효과음 원본, 중간 작업물 | 배경음악 배포 | 게임 배경음악 | 음악 아카이브 |

## 게임 개발 관점에서

- **짧은 효과음에 적합**: 총소리, 발걸음, UI 클릭음처럼 짧고 자주 재생되는 효과음은 WAV로 쓰는 게 일반적. 압축 해제 과정이 없어서 재생 지연(Latency)이 거의 없음
- **긴 배경음악에는 비적합**: 수 분짜리 BGM을 WAV로 넣으면 용량이 수십 MB → OGG나 MP3로 대체
- **Unity 권장 방식**:

```
  효과음 (짧음, 자주 재생) → WAV 또는 AIFF → Decompress on Load
  배경음악 (긺, 스트리밍)  → OGG             → Streaming
```

- **Unity 임포트 옵션**:
  - `Decompress on Load`: 메모리에 압축 해제된 채로 올려둠 → 재생 빠름, 메모리 큼 (짧은 효과음에 적합)
  - `Compressed in Memory`: 메모리에 압축된 채로 두고 재생 시 해제 → 메모리 작음, 약간의 CPU 비용
  - `Streaming`: 디스크에서 실시간으로 읽으며 재생 → 메모리 거의 안 씀 (긴 BGM에 적합)
- **원본 작업은 WAV로**: 오디오 편집 중간 작업물은 WAV로 보관하고, 최종 배포 시 OGG/MP3로 변환하는 게 표준 파이프라인

## 의문점 / 더 알아볼 것

- AIFF — Mac 계열의 WAV 대응 포맷. Unity에서 WAV와 동일하게 취급
- OGG Vorbis가 게임 업계에서 MP3보다 선호되는 이유 (라이선스 문제)
- Unity AudioClip의 Load Type이 실제 메모리 사용량에 미치는 영향
