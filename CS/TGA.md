# TGA (Targa)

## 핵심 개념

- **Truevision Graphics Adapter**의 약자. 확장자는 `.tga`
- Truevision(현 Avid)이 1984년 만든 포맷. 게임/3D 그래픽 업계에서 오래 쓰인 표준
- **무손실** 포맷이며 **알파 채널(투명도)** 을 지원
- 비압축 또는 RLE 압축 선택 가능

## 다른 포맷과 비교

| 항목                | TGA                  | PNG         | BMP         | JPG       |
| ------------------- | -------------------- | ----------- | ----------- | --------- |
| 압축                | 무압축 or RLE        | 무손실 압축 | 무압축      | 손실 압축 |
| 투명도              | 지원 (32bit)         | 지원        | 기본 미지원 | 미지원    |
| 파일 크기           | 큼 (무압축 시 BMP급) | 중간        | 매우 큼     | 작음      |
| 알파 채널 품질      | 매우 정밀            | 정밀        | -           | -         |
| 3D/게임 업계 친화성 | 매우 높음            | 높음        | 낮음        | 낮음      |

## 구조

```
┌─────────────────┐
│   File Header   │  ← 이미지 크기, 색상 깊이, 알파 여부 등
├─────────────────┤
│   Color Map     │  ← 인덱스 컬러 방식일 때만 존재
├─────────────────┤
│   Pixel Data    │  ← RGB or RGBA 픽셀값 (BMP처럼 단순 구조)
└─────────────────┘
```

- BMP처럼 구조가 단순하면서 알파 채널까지 지원하는 게 TGA의 강점
- 픽셀 데이터가 **아래에서 위로(Bottom-Up)** 저장되는 게 기본 — 일부 툴에서 뒤집혀 보이는 원인

## 색상 깊이(Bit Depth)

```
24bit = RGB (각 8bit) — 투명도 없음
32bit = RGBA (각 8bit) — 투명도 포함
```

## 게임 개발 관점에서

- **게임 업계 전통 포맷**: 3ds Max, Maya, Photoshop 등 DCC 툴에서 오래전부터 기본 지원. 언리얼 엔진이 텍스처 포맷으로 TGA를 기본으로 밀던 시절이 있었음
- **Unity에서도 임포트 지원**: PNG와 동일하게 투명도 있는 텍스처로 사용 가능. 단, 용량이 크기 때문에 현재는 PNG나 PSD에 밀리는 추세
- **알파 채널 정밀도**: 32bit TGA는 알파값이 0~255로 정밀하게 표현됨. 반투명 이펙트, 스모크, 글로우 텍스처처럼 섬세한 투명도 표현이 필요한 에셋에 적합
- **셰이더/텍스처 작업**: 노멀맵, 스페큘러맵 같은 PBR 텍스처를 TGA로 저장하는 파이프라인이 아직 많은 스튜디오에 남아있음
- **현재 위치**: 신규 프로젝트에서는 PNG나 PSD로 대체되는 추세지만, 레거시 파이프라인이나 언리얼 기반 프로젝트에서는 여전히 자주 만남

## 의문점 / 더 알아볼 것

- TGA의 Bottom-Up 저장 방식이 OpenGL 좌표계와 맞아서 오래 쓰인 이유
- PSD(포토샵 포맷)를 Unity가 직접 임포트할 수 있는 구조와 TGA의 차이
