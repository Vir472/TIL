# RAM vs VRAM

## 핵심 개념

- **RAM (Random Access Memory)**: CPU가 사용하는 주 메모리. 운영체제, 프로그램, 일반 데이터가 올라가는 공간
- **VRAM (Video RAM)**: GPU가 사용하는 전용 메모리. 텍스처, 렌더링 데이터, 딥러닝 모델 가중치 등이 올라가는 공간

## 비교

| 항목          | RAM                | VRAM                       |
| ------------- | ------------------ | -------------------------- |
| 사용 주체     | CPU                | GPU                        |
| 용량 (일반적) | 16~64GB            | 8~24GB (소비자용)          |
| 대역폭        | 낮음               | 매우 높음                  |
| 역할          | 일반 프로그램 실행 | 그래픽/AI 연산 데이터 적재 |

## AI분야에서 활용

- LLM 같은 대형 모델은 가중치 크기가 VRAM 용량을 초과하는 경우가 많음
- VRAM 부족 → 모델을 RAM에 올리거나 양자화(Quantization) 필요
- 로컬 LLM 실행 시 VRAM이 병목이 되는 이유

## 게임 개발에서의 활용

- **텍스처 메모리**: 게임의 텍스처, 메시, 셰이더는 VRAM에 올라가야 GPU가 렌더링 가능. VRAM 초과 시 RAM에서 가져오는 스트리밍 발생 → 프레임 드랍 원인
- **Unity 관점**: `Resources.Load()`로 에셋을 불러오면 RAM에 올라가고, 실제 렌더링 시 VRAM으로 업로드됨. 에셋 번들/어드레서블로 메모리 관리하는 이유
- **VRAM 최적화 기법**: 텍스처 압축(DXT, ASTC), 밉맵(Mipmap) 사용, 텍스처 아틀라스로 VRAM 사용량 줄임
- **콘솔 게임**: PS5/Xbox는 RAM과 VRAM이 통합 메모리(Unified Memory)로 구성 → PC와 메모리 구조가 다름
