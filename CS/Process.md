# 프로세스

## 핵심 개념

- **프로세스(Process)**: 실행 중인 프로그램의 인스턴스. 프로그램이 메모리에 올라가 CPU의 제어를 받는 상태
- 프로그램(정적) vs 프로세스(동적): 프로그램은 디스크에 있는 파일, 프로세스는 그게 실행된 것

## 게임 개발 관점에서

- 게임 자체가 하나의 프로세스
- 렌더링 스레드, 물리 스레드, 오디오 스레드 등이 그 안에서 멀티스레드로 동작
- Unity는 기본적으로 메인 스레드 중심이지만 Job System으로 멀티스레드 활용 가능
  | 스레드 | 역할 |
  |--------|------|
  | 메인 스레드 | 게임 로직, Update() 실행, Unity API 호출 |
  | 렌더 스레드 | GPU에 렌더링 명령 전달 |
  | 워커 스레드 | Job System으로 분배된 연산 처리 |
  | 오디오 스레드 | 사운드 믹싱 및 재생 |

- **왜 Unity가 메인 스레드 중심이냐면**: GameObject, Transform 등 Unity 오브젝트는 스레드 안전(Thread-Safe)하지 않음. 멀티스레드에서 접근하면 크래시 위험
