# BMP (Bitmap)

## 핵심 개념

- **Bitmap**의 약자. 확장자는 `.bmp`
- Microsoft가 Windows용으로 만든 이미지 포맷
- **비압축(Uncompressed)** 방식 — 픽셀 데이터를 있는 그대로 저장
- 압축을 안 하기 때문에 파일이 매우 크지만, 열고 쓰는 속도가 빠르고 품질 손실이 없음

## 저장 방식

```
픽셀 하나 = RGB 각 8bit = 24bit (3바이트)
1920x1080 이미지 = 1920 × 1080 × 3바이트 = 약 6MB (압축 없음)

같은 이미지를 JPG로 저장하면 → 수백 KB 수준
```

모든 픽셀의 색상값을 그냥 순서대로 나열해서 저장하는 구조

## 다른 포맷과 비교

| 항목      | BMP                        | JPG        | PNG            |
| --------- | -------------------------- | ---------- | -------------- |
| 압축      | 없음                       | 손실 압축  | 무손실 압축    |
| 파일 크기 | 매우 큼                    | 작음       | 중간           |
| 품질      | 완전 무손실                | 손실 있음  | 무손실         |
| 투명도    | 기본 미지원                | 미지원     | 지원           |
| 처리 속도 | 빠름 (파싱 단순)           | 중간       | 중간           |
| 용도      | 중간 작업물, 윈도우 시스템 | 사진, 배경 | UI, 스프라이트 |

## 구조

BMP 파일은 크게 세 부분으로 구성됨

```
┌─────────────────┐
│   File Header   │  ← 파일 크기, 포맷 식별자("BM") 등 메타데이터
├─────────────────┤
│   Info Header   │  ← 이미지 가로/세로 크기, 색상 깊이(bit depth) 등
├─────────────────┤
│   Pixel Data    │  ← 실제 픽셀 색상값 (압축 없이 그대로)
└─────────────────┘
```

구조가 단순해서 파서 구현이 쉬움 → 이미지 포맷 공부할 때 BMP부터 시작하는 이유

## 게임 개발 관점에서

- **실제 게임 에셋으로는 거의 안 씀**: 파일이 너무 커서 배포용 게임에 넣기 부적합. Unity도 BMP 임포트는 지원하지만 권장하지 않음
- **중간 작업물 저장**: 이미지 편집 중간 단계를 저장할 때 품질 손실 없이 보관 가능. 단, PNG가 무손실이면서 압축도 되므로 사실상 PNG에 밀림
- **픽셀 데이터 직접 접근**: 구조가 단순해서 코드로 픽셀을 직접 읽고 쓸 때 다루기 쉬움. 커스텀 이미지 처리 툴이나 셰이더 디버깅 시 원본 픽셀값 확인용으로 활용
- **Windows API와의 친화성**: Win32 API나 GDI+에서 기본 이미지 포맷으로 BMP를 사용. Windows 전용 툴 개발 시 여전히 만남

## 의문점 / 더 알아볼 것

- BMP도 RLE(Run-Length Encoding) 압축을 선택적으로 지원하는데, 실제로 쓰이는지
- 비압축 포맷임에도 BMP가 PNG보다 파일이 클 수 있는 이유 (PNG의 압축 효율)
