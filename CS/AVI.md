# AVI (Audio Video Interleave)

## 핵심 개념

- **Audio Video Interleave**의 약자. 확장자는 `.avi`
- Microsoft가 1992년 Windows용으로 만든 동영상 컨테이너 포맷
- **컨테이너(Container)** 포맷 — 영상과 오디오 데이터를 하나의 파일에 담는 틀
- 내부 코덱은 자유롭게 선택 가능 (압축 방식을 AVI가 직접 결정하지 않음)

## 컨테이너 vs 코덱

```
컨테이너 (AVI, MP4, MKV)
  └─ 영상 트랙 → 코덱으로 압축된 영상 데이터 (H.264, H.265, DivX 등)
  └─ 오디오 트랙 → 코덱으로 압축된 오디오 데이터 (MP3, AAC, PCM 등)
  └─ 메타데이터 → 재생 시간, 해상도, 자막 등
```

같은 AVI 파일이라도 내부 코덱이 다르면 재생이 안 될 수 있는 이유

## 파일 구조 — RIFF 기반

WAV와 동일하게 RIFF 청크 구조를 사용

```
┌──────────────┐
│  RIFF 헤더   │  ← "AVI " 식별자
├──────────────┤
│  hdrl 청크   │  ← 영상/오디오 포맷 정보 (헤더)
├──────────────┤
│  movi 청크   │  ← 실제 영상·오디오 데이터 (인터리브 방식)
├──────────────┤
│  idx1 청크   │  ← 인덱스 (탐색용)
└──────────────┘
```

**인터리브(Interleave)**: 영상 프레임과 오디오 데이터를 번갈아가며 저장

```
[영상 프레임1][오디오][영상 프레임2][오디오][영상 프레임3]...
```

→ 순차 재생에 최적화. 중간 탐색(Seek)이 느린 단점

## 다른 동영상 포맷과 비교

| 항목        | AVI              | MP4      | MKV         | WebM   |
| ----------- | ---------------- | -------- | ----------- | ------ |
| 제작사      | Microsoft        | MPEG     | 오픈소스    | Google |
| 컨테이너    | RIFF 기반        | ISO 기반 | Matroska    | WebM   |
| 코덱 유연성 | 높음             | 높음     | 매우 높음   | 제한적 |
| 스트리밍    | 부적합           | 적합     | 부적합      | 적합   |
| 파일 크기   | 큼 (비압축 가능) | 작음     | 중간        | 작음   |
| 자막 내장   | 제한적           | 가능     | 매우 유연   | 가능   |
| 현재 활용도 | 레거시           | 표준     | 고화질 보관 | 웹     |

## AVI의 한계

- **4GB 파일 크기 제한**: RIFF 구조 특성상 인덱스가 32bit → 4GB 초과 불가 (OpenDML 확장으로 일부 해결)
- **스트리밍 부적합**: 인덱스가 파일 끝에 있어서 네트워크 스트리밍에 불리
- **자막/챕터 지원 빈약**: MP4, MKV 대비 메타데이터 기능 부족
- 이런 한계로 현재는 MP4에 밀려 레거시 포맷 취급

## 게임 개발 관점에서

- **게임 에셋으로는 거의 안 씀**: 인게임 동영상(컷신)은 주로 MP4(H.264) 또는 Unity의 WebM 포맷 사용. AVI는 용량이 크고 스트리밍에 불리해서 부적합
- **Unity 동영상 지원 포맷**: Unity의 Video Player 컴포넌트는 MP4, WebM, MOV를 주로 지원. AVI는 플랫폼에 따라 지원 여부가 다름
- **캡처/레코딩 중간 포맷**: 게임 개발 중 화면 캡처나 테스트 녹화 시 비압축 AVI로 저장 후 MP4로 변환하는 파이프라인이 일부 남아있음
- **컷신 파이프라인**: 고품질 컷신은 외부에서 MP4로 제작 → Unity VideoPlayer로 재생하거나, 짧은 컷신은 Unity Timeline으로 실시간 렌더링

## 의문점 / 더 알아볼 것

- H.264와 H.265(HEVC)의 압축 효율 차이 — 게임 컷신 품질에 미치는 영향
- Unity VideoPlayer의 스트리밍 방식과 메모리 로드 방식의 차이
